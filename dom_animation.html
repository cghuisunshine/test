DOM Editorial Engine

A single-file DOM reimplementation of the “text flowing around draggable circles” idea. This version uses real DOM measurement, absolute-positioned line boxes, and per-row obstacle calculations instead of a custom canvas/typesetting engine.

Drag the circles. Each frame, the script computes blocked horizontal intervals for each text row, subtracts them from the available width, then fits words into the remaining slot with a hidden measuring node.

How it works: for each row, compute circle intersections, merge blocked intervals, derive free segments, then fit as many words as possible.
DOM tradeoff: this is easy to inspect in DevTools, but repeated measurement can be more expensive than a custom zero-read layout engine.
Core APIs: offsetWidth, requestAnimationFrame, absolute positioning, pointer events.
