1) What HTML is

HTML (HyperText Markup Language) is the language that describes the structure and meaning of content on the web. It is not a programming language: HTML does not have variables and loops. Instead, it marks content as "this is a heading", "this is a link", "this is a form field", etc.

Think of a web page as a team: HTML = structure/content, CSS = design/layout, JavaScript = behavior/interaction.

How browsers use HTML

  1. The browser downloads HTML.
  2. It parses it into a tree called the DOM (Document Object Model).
  3. It downloads CSS and applies styles.
  4. It runs JavaScript, which can change the DOM.

2) Origin and history (where HTML came from)

HTML originated as part of the early World Wide Web. It was designed so scientists and researchers could share documents that contained hypertext links (clickable references) across the internet.

Key milestones (high level)

  • 1989-91: Tim Berners-Lee proposes the Web at CERN and builds the first browser/editor and server.
  • Early 1990s: HTML evolves from SGML-inspired ideas: simple tags for headings, paragraphs, links.
  • 1995: HTML 2.0 helps standardize common browser behavior.
  • Late 1990s: HTML 4.01 adds stronger support for scripting, forms, and internationalization.
  • 2000s: XHTML explores stricter XML rules; adoption is mixed on the open web.
  • 2014+: HTML5 becomes the foundation for modern web apps: semantic elements, audio/video, richer forms.
  • Today: HTML is maintained as a living standard (continuously updated) alongside related specs.

"HTML5" is often used as a broad term for modern web features. In practice, the web is a collection of specifications: HTML, CSS, JavaScript, DOM APIs, and many platform APIs.

Why the origin matters

HTML was built for documents first. Modern apps can be built with it too, but the best pages still respect the original goal: clear meaning (semantics), easy navigation (links), and broad access (works for everyone).

3) Tags, elements, and attributes

An element usually consists of an opening tag, content, and a closing tag. Some elements are void (no closing tag), like <img>.

Example

<p class="lead">Hello, web!</p>
  • p is the tag name
  • class is an attribute name
  • lead is the attribute value

Common attributes

Attribute Where used What it does
id Most elements Unique identifier (also used for linking to a section like #about)
class Most elements Groups elements for CSS styling and JS selection
href <a> Link destination
src <img>, <script> Resource URL
alt <img> Text alternative for images (critical for accessibility)

Nesting rules (quick idea)

Some elements can contain many other elements (like <div>), while some have restrictions. For example, a <p> should not contain another block-level element like <div>.

4) Basic document skeleton

Every HTML page starts with a doctype and has two main sections: <head> (metadata) and <body> (visible content). 

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello</h1>
  </body>
</html>

The lang attribute helps screen readers and search engines understand the language.

5) Headings, paragraphs, and text

Headings

Use headings to create a meaningful outline. Typically a page has one main <h1>, then sections use <h2>, sub-sections use <h3>, and so on. 

<h1>Solar System</h1>
<h2>Planets</h2>
<h3>Earth</h3>

Paragraphs and emphasis

<p>A paragraph is written with the <code>p</code> element.</p>
<p>Use <strong>strong</strong> for importance and <em>em</em> for emphasis.</p>

Inline text elements

Element Use Example
<span> Generic inline container (usually for styling) <span class="tag">New</span>
<code> Inline code Use <code>href</code>
<br> Line break (use rarely) Line 1<br>Line 2
<hr> Thematic break between topics <hr>

6) Semantic layout (meaningful structure)

Semantic HTML uses elements that describe what the content is, not just how it looks. This improves accessibility, SEO, and maintainability.

Common semantic elements

Element Meaning Typical use
<header> Intro for a page/section Logo, title, navigation
<nav> Navigation links Main menu, table of contents
<main> Main content The primary topic of the page (usually one)
<section> Section of related content Chapters/topics
<article> Self-contained piece Blog post, news article, forum post
<aside> Related but not main Sidebar, callouts
<footer> Closing info Author, links, copyright notice

Example page layout

<header>
  <h1>City Guide</h1>
  <nav>
    <a href="#food">Food</a>
    <a href="#parks">Parks</a>
  </nav>
</header>

<main>
  <section id="food">
    <h2>Food</h2>
    <article>
      <h3>Street Market</h3>
      <p>...</p>
    </article>
  </section>
</main>

<footer>
  <p>Updated 2025</p>
</footer>

Do not use <div> for everything. Use semantic elements first, then use <div> only when there is no meaningful element.

7) Links and navigation

Links are the original superpower of the Web. The anchor element <a> creates a hyperlink.

Link to another page

<a href="https://example.com">Visit Example</a>

Link to a section on the same page

<a href="#forms">Jump to forms</a>

<section id="forms">...</section>

Open in a new tab (and stay safe)

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Open Example</a>

Use clear link text. "Click here" is vague; "Download the syllabus (PDF)" is better.

8) Images, audio, and video

Images

<img src="images/mountain.jpg" alt="A snowy mountain under a blue sky" width="900" height="600" />

Always write meaningful alt text. If an image is purely decorative, use alt="" (empty) so screen readers can skip it.

Figure and caption

<figure>
  <img src="chart.png" alt="Bar chart of monthly sales" />
  <figcaption>Figure 1: Monthly sales in 2025.</figcaption>
</figure>

Audio

<audio controls>
  <source src="song.mp3" type="audio/mpeg" />
  <p>Your browser does not support audio.</p>
</audio>

Video

<video controls width="720">
  <source src="movie.mp4" type="video/mp4" />
  <p>Your browser does not support video.</p>
</video>

9) Lists

Lists make content scannable and are extremely common in navigation, instructions, and summaries.

Unordered list (bullets)

<ul>
  <li>Milk</li>
  <li>Bread</li>
  <li>Eggs</li>
</ul>

Ordered list (numbers)

<ol>
  <li>Preheat oven</li>
  <li>Mix ingredients</li>
  <li>Bake</li>
</ol>

Description list (terms and definitions)

<dl>
  <dt>HTTP</dt>
  <dd>The protocol used to transfer web resources.</dd>
  <dt>URL</dt>
  <dd>The address of a resource on the web.</dd>
</dl>

10) Tables

Tables are for tabular data (data in rows and columns). Do not use tables for page layout.

Basic table

<table>
  <caption>Bus schedule</caption>
  <thead>
    <tr>
      <th scope="col">Route</th>
      <th scope="col">First</th>
      <th scope="col">Last</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">12A</th>
      <td>06:10</td>
      <td>22:40</td>
    </tr>
  </tbody>
</table>

Use <caption> and proper header cells (<th> with scope) for accessibility.

11) Forms

Forms collect user input. HTML provides many input types and built-in validation. Always pair inputs with <label>.

Simple form example

<form action="/signup" method="post">
  <label for="email">Email</label>
  <input id="email" name="email" type="email" required />

  <label for="password">Password</label>
  <input id="password" name="password" type="password" minlength="8" required />

  <button type="submit">Create account</button>
</form>

Common input types

Type Use Notes
text General text Add maxlength, pattern, and helpful autocomplete
email Email address Basic validation is built-in
password Passwords Use autocomplete like current-password or new-password
number Numeric values Supports min, max, step
date Dates Browser provides a date picker
file Upload files Use accept and remember server-side validation

Grouping fields

<fieldset>
  <legend>Contact preferences</legend>
  <label><input type="radio" name="contact" value="email" /> Email</label>
  <label><input type="radio" name="contact" value="sms" /> SMS</label>
</fieldset>

HTML validation improves user experience but is not security. Always validate and sanitize on the server too.

12) Metadata and SEO basics

Metadata lives in <head>. It helps browsers, social apps, and search engines understand your page.

Essential head tags

<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>Page title</title>
  <meta name="description" content="A short summary of this page." />
</head>

Favicon

<link rel="icon" href="/favicon.ico" />

Open Graph (social previews)

<meta property="og:title" content="My Article" />
<meta property="og:description" content="A short summary..." />
<meta property="og:image" content="https://example.com/preview.png" />

Good SEO starts with good content and semantic HTML: real headings, descriptive links, accessible images.

13) Accessibility (a11y)

Accessibility means people with different abilities can use your page: screen readers, keyboard users, low-vision users, and more.

High-impact habits

  • Use semantic elements (<button> for actions, <a> for navigation).
  • Ensure everything works with a keyboard (use Tab, Enter, Space).
  • Provide text alternatives: alt on images, captions for videos when possible.
  • Use proper labels for form controls (do not rely on placeholder text).
  • Maintain good color contrast.

ARIA: use when needed

ARIA attributes can help describe UI controls, but they are not a replacement for semantic HTML. Prefer native elements first. 

<button aria-label="Close dialog">
  ×
</button>

Avoid making clickable <div> or <span> elements. Use a real <button>.

14) Responsive basics

Responsive design means your page adapts to different screen sizes. HTML helps by being flexible by default, and CSS handles most of the layout.

Viewport meta tag (required)

<meta name="viewport" content="width=device-width, initial-scale=1" />

Responsive images (idea)

You can provide multiple image sizes with srcset so phones download smaller files. 

<img
  src="photo-800.jpg"
  srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw, 600px"
  alt="A lighthouse at sunset"
/>

15) Modern HTML features (useful, no frameworks needed)

Details/summary (built-in accordion)

<details>
  <summary>What is HTML?</summary>
  <p>HTML is a markup language for structuring web content.</p>
</details>

Dialog (native modal)

Dialog needs JavaScript to open/close, but the element and accessibility behavior are built-in.

<dialog id="info">
  <h2>About</h2>
  <p>This is a dialog.</p>
  <form method="dialog">
    <button>Close</button>
  </form>
</dialog>

Template (inert HTML for later use)

<template id="card-template">
  <article class="card">
    <h2>Title</h2>
    <p>Body text...</p>
  </article>
</template>

Even if you use frameworks, knowing these built-ins helps you build simpler pages and debug faster.

16) Best practices (quick checklist)

  • Start semantic: choose elements based on meaning, then style with CSS.
  • Keep headings ordered: do not jump from h2 straight to h5 without reason.
  • Write accessible forms: always use <label>, fieldset, and clear errors.
  • Optimize media: use appropriate image sizes, and include width/height to reduce layout shift.
  • Validate and test: open the page in multiple browsers, and use keyboard-only navigation.

Avoid deprecated tags like <font> and using HTML for visual styling. Use CSS instead.
