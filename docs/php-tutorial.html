1) How PHP Works

PHP is a server-side programming language. That means PHP code runs on the server, generates HTML (and sometimes JSON), and the browser receives the output.

Mental model

  1. Browser requests a URL (e.g., /profile.php).
  2. Web server runs the PHP file.
  3. PHP produces HTML.
  4. Browser renders the HTML.

Key idea

Users never “download your PHP code”. They only see the output. If PHP code appears in the browser, the server is misconfigured.

2) Setup & Running

You can run PHP locally either with a full web stack (Apache/Nginx + PHP) or with PHP’s built-in dev server.

Option A: Built-in dev server (recommended for learning)

# In your project folder
php -S localhost:8000

Then open http://localhost:8000. Put a file like index.php in the folder.

Option B: XAMPP / MAMP / WAMP

These bundles install a local Apache server + PHP (and often MySQL). Place your project in the “htdocs” folder (location depends on the bundle), then browse to your local site.

3) Syntax Basics

Hello world

<!doctype html>
<html>
  <body>
    <?php echo "Hello, PHP!"; ?>
  </body>
</html>

PHP tags

  • <?php ... ?> is the standard tag.
  • Statements usually end with ;.
  • Comments: //, #, or /* ... */.

Echo vs print

Keyword What it does Typical use
echo Outputs one or more strings Most common for HTML output
print Outputs a string (returns 1) Rare; echo is more common

4) Variables & Types

Variables start with $. PHP is dynamically typed: variables can change type. 

<?php
$name = "Asha";   // string
$age = 16;        // int
$height = 1.65;   // float
$isStudent = true; // bool

echo $name;
?>

Common types

  • string, int, float, bool
  • array, object, null

Type + value inspection

<?php
$x = "42";
var_dump($x); // string(2) "42"
?>

Don’t use var_dump() on production pages. It can leak sensitive information.

5) Strings

Single vs double quotes

<?php
$name = "Kai";
echo 'Hello $name';   // prints: Hello $name
echo "Hello $name";   // prints: Hello Kai
?>

Double quotes allow variable interpolation and escape sequences like \n.

Concatenation

<?php
$first = "Ada";
$last = "Lovelace";
echo $first . " " . $last;
?>

Helpful string functions

Function Example Result
strlen strlen("abc") 3
strtolower strtolower("Hi") "hi"
trim trim(" hi ") "hi"
substr substr("hello", 1, 3) "ell"

6) Control Flow

If / else

<?php
$score = 78;
if ($score >= 80) {
  echo "A";
} elseif ($score >= 70) {
  echo "B";
} else {
  echo "Needs improvement";
}
?>

Loops

for

<?php
for ($i = 1; $i <= 5; $i++) {
  echo $i;
}
?>

while

<?php
$i = 1;
while ($i <= 5) {
  echo $i;
  $i++;
}
?>

Switch

<?php
$day = "Mon";
switch ($day) {
  case "Mon":
    echo "Start";
    break;
  case "Fri":
    echo "Almost weekend";
    break;
  default:
    echo "Regular day";
}
?>

7) Arrays

PHP arrays can be indexed (list-like) or associative (key/value).

Indexed arrays

<?php
$colors = ["red", "green", "blue"];
echo $colors[0]; // red
?>

Associative arrays

<?php
$student = [
  "name" => "Lina",
  "grade" => 12,
  "club" => "Robotics",
];
echo $student["name"];
?>

foreach (most useful loop for arrays)

<?php
foreach ($student as $key => $value) {
  echo $key . ": " . $value . "<br>";
}
?>

Common array helpers

Goal Function Example
Add to end $arr[] = ... / array_push $colors[] = "purple";
Count items count count($colors)
Sort sort / rsort sort($colors)
Get keys array_keys array_keys($student)

8) Functions

Define and call

<?php
function greet(string $name): string {
  return "Hello, " . $name;
}

echo greet("Sam");
?>

Type hints (string, int, etc.) help readability, but PHP can still accept different types unless you enable strict typing.

Default parameters

<?php
function power(int $base, int $exp = 2): int {
  return $base ** $exp;
}

echo power(5);    // 25
echo power(2, 3); // 8
?>

Useful built-ins

  • date, json_encode, json_decode
  • filter_var (validation)
  • password_hash, password_verify (passwords)

9) Superglobals

Superglobals are built-in arrays available everywhere. They commonly carry request data.

Name Contains Example use
$_GET Query parameters /page.php?name=Kai
$_POST Form body parameters Login forms
$_SERVER Request/server metadata $_SERVER["REQUEST_METHOD"]
$_SESSION Session storage Remember logged-in user
$_COOKIE Cookies sent by browser Theme preference
$_FILES Uploaded files Profile photo upload

Never trust user input. Treat everything from $_GET, $_POST, and $_COOKIE as untrusted. Always validate and escape.

10) Forms (GET/POST)

Forms send user input to a server endpoint (often another PHP file). Use method="post" for sensitive data.

Example: simple POST form + handler

form.html

<form action="submit.php" method="post">
  <label>Name: <input name="name" /></label>
  <button type="submit">Send</button>
</form>

submit.php

<?php
function h(string $s): string {
  return htmlspecialchars($s, ENT_QUOTES, "UTF-8");
}

$name = $_POST["name"] ?? "";
$name = trim($name);

if ($name === "") {
  echo "Name is required.";
  exit;
}

echo "Hello " . h($name);
?>

htmlspecialchars() prevents XSS by escaping special characters before outputting into HTML.

GET example

<?php
$q = $_GET["q"] ?? "";
echo "You searched for: " . htmlspecialchars($q, ENT_QUOTES, "UTF-8");
?>

11) Sessions & Cookies

Sessions (server-side)

<?php
session_start();

// Set a value
$_SESSION["username"] = "Kai";

// Read it later
echo $_SESSION["username"] ?? "Guest";
?>

Sessions usually use a session cookie to link the browser to server-side session data.

Cookies (client-side)

<?php
setcookie("theme", "dark", [
  "expires" => time() + 60 * 60 * 24 * 30,
  "path" => "/",
  "secure" => true,
  "httponly" => true,
  "samesite" => "Lax",
]);
?>

Cookies can be modified by the client. Never store secrets (like passwords) in cookies.

12) Files & Uploads

Reading and writing files

<?php
$path = __DIR__ . "/notes.txt";
file_put_contents($path, "Hello\n", FILE_APPEND);
$contents = file_get_contents($path);
echo nl2br(htmlspecialchars($contents, ENT_QUOTES, "UTF-8"));
?>

Use __DIR__ to build safe paths. Escape output if you display file contents in HTML.

Uploads (basic pattern)

<!-- upload.html -->
<form action="upload.php" method="post" enctype="multipart/form-data">
  <input type="file" name="photo" />
  <button type="submit">Upload</button>
</form>
<?php
// upload.php
if (!isset($_FILES["photo"])) { exit("No file"); }

$f = $_FILES["photo"];
if ($f["error"] !== UPLOAD_ERR_OK) { exit("Upload failed"); }

// Always validate type/size and choose your own filename.
if ($f["size"] > 2_000_000) { exit("Too large"); }

$targetDir = __DIR__ . "/uploads";
if (!is_dir($targetDir)) { mkdir($targetDir); }

$target = $targetDir . "/" . bin2hex(random_bytes(16)) . ".bin";
move_uploaded_file($f["tmp_name"], $target);

echo "Uploaded.";
?>

Uploads are a common attack vector. Never trust the original filename, and never allow arbitrary uploads into a web-executable folder.

13) OOP Basics

Object-oriented programming (OOP) helps organize larger projects. In PHP, classes define objects with properties and methods.

Class example

<?php
class Student {
  public string $name;
  private int $grade;

  public function __construct(string $name, int $grade) {
    $this->name = $name;
    $this->grade = $grade;
  }

  public function getGrade(): int {
    return $this->grade;
  }
}

$s = new Student("Lina", 12);
echo $s->name;
echo $s->getGrade();
?>

Inheritance (quick idea)

<?php
class User { public function role(): string { return "user"; } }
class Admin extends User { public function role(): string { return "admin"; } }
?>

Use inheritance when one class is truly a “special case” of another. Otherwise consider composition.

14) Databases (PDO + Prepared Statements)

For SQL databases, PHP commonly uses PDO. The most important rule: use prepared statements to prevent SQL injection.

Connect to MySQL

<?php
$dsn = "mysql:host=localhost;dbname=school;charset=utf8mb4";
$user = "root";
$pass = "";

$pdo = new PDO($dsn, $user, $pass, [
  PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
  PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
]);
?>

Safe query (prepared statement)

<?php
$id = $_GET["id"] ?? "0";

$stmt = $pdo->prepare("SELECT id, name, grade FROM students WHERE id = :id");
$stmt->execute(["id" => $id]);
$row = $stmt->fetch();

if (!$row) {
  echo "Not found";
  exit;
}

echo htmlspecialchars($row["name"], ENT_QUOTES, "UTF-8");
?>

Never build SQL like "... WHERE id = " . $_GET["id"]. Even if you “trust” the value.

15) Security Checklist (Essential Habits)

Output escaping (XSS prevention)

Any time you output user-controlled data into HTML, escape it:

<?php
function h(string $s): string {
  return htmlspecialchars($s, ENT_QUOTES, "UTF-8");
}
echo h($userInput);
?>

Input validation

<?php
$email = $_POST["email"] ?? "";
if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
  exit("Invalid email");
}
?>

Password storage

<?php
$hash = password_hash($password, PASSWORD_DEFAULT);
if (password_verify($passwordAttempt, $hash)) {
  echo "OK";
}
?>

CSRF protection (idea)

For forms that change server state (creating posts, updating profiles), add a CSRF token: store a random token in $_SESSION and include it as a hidden form field, then verify it on submit.
