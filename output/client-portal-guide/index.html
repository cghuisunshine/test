Login & Dashboard

The client portal is the main workspace for strata managers and property representatives to complete reserve-planning data collection. After sign-in, the dashboard becomes the home base for the rest of the submission process.

  1. Open the portal URL and sign in with the assigned email and password.
  2. If a password has been forgotten, use Forgot Password? from the login page before attempting again.
  3. After login, review the dashboard progress summary for survey questions, documents, and timelines so you can see what is still incomplete.
  4. Use the left navigation to move between Dashboard, Strata Information, Strata Members, Timelines, Survey, Documents, and Inspection Date when that final step becomes available.
  5. Use the dashboard priority-task cards to jump directly into the next required action, including survey completion, document uploads, and deadline review.
  6. Remember that the inspection-date workflow stays locked until survey answers, documents, and deadlines have all been completed first.

No Active Service

If the portal shows a No Active Service message, the strata does not currently have an active file request attached to the account. Because depreciation-report work is tied to a specific file number, the client workspace remains limited until activation is approved.

  1. If you see the inactive-service message across multiple pages, request activation from the Strata Reserve Planning team.
  2. Wait for the activation request to be reviewed. The guide states that review typically takes three to five business days.
  3. If the request is denied, read the explanation in the notification, dismiss it, and submit a corrected activation request when ready.
  4. Once the request is approved, return to the dashboard and continue with the full client workflow.

Strata Information & Strata Members

These pages show both the building-level profile and the people associated with the strata. Accuracy here matters because these details affect appointment filtering, document requirements, question visibility, and legal reporting context.

  1. Open Strata Information to review the building profile, including strata plan, address, property types, and other core strata details.
  2. If any building information is wrong, contact clientcare@stratareserveplanning.com to request an update.
  3. Confirm that the listed property types are correct, because they determine which questions and documents apply to the file.
  4. Open Strata Members to review the individuals associated with the strata, including roles, accessible sections, and contact details.
  5. Use the available update action to keep your personal contact details and password current so the SRP team can reach the right person during the report process.

Timelines

The Timelines page captures the key dates needed to plan and deliver the depreciation report. Some values may already be pre-filled, but clients are expected to confirm and complete what is missing.

  1. Open Timelines from the left navigation.
  2. Enter or confirm the current fiscal year start date, the date of the last AGM, the date of the last depreciation report, and the target date if one applies.
  3. If this is the first cycle, use No AGM to date or No report to date as appropriate.
  4. Make sure AGM and prior-report dates are in the past, because future dates are not allowed.
  5. Use a target date that leaves enough lead time. The guide states that it must be at least 30 days in the future.
  6. Finalize the timelines only after careful review, because the timeline values lock after submission and later changes require manual help from Strata Reserve Planning.
Important: If no target date is selected, the guide explains that the system can calculate a default based on the fiscal year timeline.

Answering Surveys

The survey flow starts on an overview page and then moves into detailed question pages grouped by property type. Top-level questions are mandatory, while many follow-up prompts are conditional.

  1. Open Survey or use the dashboard shortcut to access the question set for the file.
  2. Review the overview page to see which survey sections still need work, and use Download Survey if an empty form is needed for offline reference.
  3. Open a section and answer every top-level question with either a direct answer, Not Available, or Not Applicable.
  4. Answer follow-up questions where possible. The guide notes that many follow-up questions can remain blank if the information is not available.
  5. Use Save Changes to keep progress for later, or Save and Download PDF if a completed copy is needed for review or printing.
  6. Return to Overview at any time to see what remains, then finalize only when all required answers are complete because the survey becomes read-only after final submission.
Manual option: The transcript also describes downloading the PDF, completing it by hand, and mailing it to the Vancouver office if a paper workflow is needed.

Document Uploads

The Documents page is where clients upload requested files for the active report. It supports multiple document versions, autosave behavior, and later review by the SRP team.

  1. Open Documents to review the requested document groups that apply to your strata and accessible property types.
  2. Expand each requested item and upload the appropriate file, or mark it as Not Available or Not Applicable when that is the correct status.
  3. Upload as many requested versions as possible when the same document type is needed for multiple years or reporting cycles.
  4. Rely on autosave if you need to stop partway through. The guide notes that uploads remain in place so co-workers can return and continue later.
  5. Finalize documents only when you are satisfied with the submission, because further changes are blocked unless a document is rejected during review.
  6. If a document is rejected, read the admin note, dismiss the message, then upload a replacement file with a different filename.
Next step: Once documents, survey answers, and timelines are complete and accepted, the inspection-booking workflow becomes available.

Appointment Booking

Appointment booking becomes available only after survey answers, documents, and timelines have been completed and accepted. The booking flow covers both the physical inspection and the later draft meeting.

  1. Once the workflow unlocks, open Inspection Date from the dashboard or navigation to access the booking calendar.
  2. Review the available dates for the assigned inspector and note the yellow reference dates that mark approved-document timing and target completion timing.
  3. Select two different dates as your first and second choices, then add any special notes if needed before submitting the request.
  4. After submission, the request stays pending and the chosen dates cannot be edited until the SRP team reviews them.
  5. If the request is not accepted, return to the booking page, choose new first and second dates, and submit another request.
  6. Once the inspection is approved and later completed, the workflow shifts to Draft Meeting, where virtual meeting times become available and are booked in a similar review-and-confirm pattern.
Draft meetings: The transcript notes that draft-meeting availability is virtual and offered in fixed evening times, with the appointment locking again while the request is under review.
