Login & Dashboard

The client portal starts on the login page. After signing in, the dashboard becomes the main home base for the rest of the data collection process.

  1. Open the portal URL and sign in with the assigned email and password.
  2. If a password has been forgotten, use Forgot Password? from the login page before attempting again.
  3. After login, review the Your Progress panel to see how many questions, documents, and timeline items are complete.
  4. Use the three main dashboard cards to jump straight into Survey, Documents, or Draft Meeting.
  5. Use the left navigation for direct access to every client section.

Strata Information & Strata Members

These two pages show the building profile and the client member record associated with the strata. They are reference pages first, with update actions where permitted.

  1. Open Strata Information to confirm the property profile details such as strata plan, address, city, legal type, and property types.
  2. If any building details are incorrect, treat the page as a verification screen and contact the support address from the client guide rather than assuming you can edit the building profile directly.
  3. Open Strata Members to review the member record tied to the logged-in user.
  4. Use Update Your Details when personal contact details or password information needs to be changed.
  5. Use Request Section Change if the strata section allocation is wrong for the logged-in member.

Timelines

The timelines page collects key dates needed to plan and deliver the depreciation report request.

  1. Open Timelines from the left navigation.
  2. Confirm the current fiscal year start date and provide the date of the last AGM, or check No AGM to date if that applies.
  3. Provide the date of the last depreciation report, or check No report to date when no earlier report exists.
  4. Optionally enter a target date if the strata has one.
  5. Select Calculate Timelines after reviewing the dates carefully, because the transcript notes that these dates can lock after calculation.
Important: The client guide notes that AGM and prior report dates must be in the past, while the target date must leave enough lead time for the SRP team to complete the report.

Answering Surveys

The survey flow starts with an overview of required sections, then continues into detailed section-by-section questionnaires.

  1. Open Survey to review the required survey categories.
  2. Use Download Survey if a copy is needed for offline review.
  3. Select a survey category to open its questions.
  4. Answer each top-level question or use the applicable availability options described in the transcript guidance.
  5. Continue page by page until all required sections are complete, then return to the overview to confirm completion status.
Important: The transcript explains that top-level questions need an answer state before final completion, while some follow-up questions may remain blank if the information is not available.

Document Uploads

The documents area is organized by strata section and then by requested document type. Each requested document can be uploaded or marked unavailable.

  1. Open Documents to see the document groups that apply to the strata.
  2. Expand a requested item to reveal the available actions.
  3. Use Upload to add the file, or mark it as Not Available or Not Applicable when appropriate.
  4. Repeat this for each requested document in every visible strata section.
  5. Use Finalize and Submit only after reviewing all requested files, because the client guide indicates that changes become restricted after final submission unless something is rejected.
Important: The transcript notes that files are saved automatically while working through the page and that rejected files may need replacement later.

No Active Service

This flow is described in the local client guide, but it is not exposed in the active demo account used for live capture. The account available for screenshots already has an active file and lands in the standard client workspace.

  1. If the portal shows a no-active-service message, request activation from the Strata Reserve Planning team.
  2. Wait for the request to be reviewed and approved by the team.
  3. If the request is declined, review the reason shown in the notification and submit a corrected request.
  4. Once activated, return to the normal dashboard and resume the standard client workflow.
Screenshot note: No fresh screenshot is included here because this demo account does not expose the inactive-file state.

Appointment Booking

The local guide describes appointment booking after submission review. In the refreshed live demo deployment, this client account no longer exposes the older draft meeting scheduler and instead surfaces deadline follow-up through the dashboard and timelines page.

  1. Use the dashboard task cards to jump into the active next step for the file.
  2. Review the Deadlines card for status prompts and then open the timelines page for date-related follow-up.
  3. Confirm the current fiscal year, AGM, depreciation report, and target date details shown for the file.
  4. After updates are submitted, monitor dashboard notifications and timeline status messaging for follow-up requests.
  5. If scheduling returns as a visible workflow in a later account state, continue from the client dashboard into that appointment-specific route.
Demo account note: The transcript references inspection and draft meeting booking phases, but the current live client account used for capture now lands in a post-submission state with deadline tracking instead of the older scheduling calendar.
