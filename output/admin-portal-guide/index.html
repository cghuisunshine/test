Login & Dashboard

The admin portal is the operational hub for managing Strata clients, inspections, documents, and depreciation-report progress across the full reserve-planning workflow.

  1. Open the portal URL and sign in with the admin email and password.
  2. After login, confirm you are in the admin workspace and review the main metric cards for active requests, pending approvals, appointments this week, and active Strata plans.
  3. Use the Shortcuts panel for frequent actions such as creating a strata, creating a user, scheduling an appointment, uploading a document, or adding inspector setup details.
  4. Review the Urgent Actions area for canceled inspections, rebooking needs, and other items that require immediate follow-up.
  5. Use the Upcoming Appointments, Target Dates, and Recent Activity blocks to monitor what is approaching and what has changed most recently.
  6. Use the left navigation to move into the admin modules for strata, users, documents, questions, appointments, timelines, and settings.

Strata Management

The Strata page is the master record for each plan. It controls core building details, property-type setup, location-based scheduling behavior, and whether inactive plans are visible.

  1. Open Strata from the left navigation.
  2. Use the filters for strata name, strata plan, property type, and location to narrow the list quickly.
  3. Enable Show Inactive when you need to expose strata without an active file.
  4. Use Create New Strata to add a new record, completing the required plan number, address, municipality, province, postal code, country, legal type, and management-company details where applicable.
  5. Select all relevant property types because property-type selections drive question filtering and per-user access permissions later in the workflow.
  6. Set the correct location because appointment availability uses location rules when same-day inspections need admin approval.
  7. Edit existing strata records whenever maintenance is needed. The fiscal year start date is optional and can also be supplied later by the client.
  8. Delete a strata only when you are certain, because deleting it removes associated files and survey answers.

Activating a File Number

A strata becomes active once a file is created. Until that happens, the client sees an empty dashboard and several admin workflows remain unavailable.

  1. Create a file manually with Create New File when the admin team is initiating the request.
  2. Alternatively, review client-submitted activation requests from the dashboard and approve or deny them.
  3. When approving a request, assign the file number and add any note needed for client context if the request needs clarification or is denied.
  4. After file creation, configure the required document list, including versioning and tags where needed.
  5. Configure survey questions based on the strata's selected property types, and treat those changes as file-specific rather than global defaults.
  6. Reorder questions within the file setup when needed, but make category changes only in the global Questions module.

Strata Active File

Once a strata has an active file, its detail view becomes the request-level control center for answers, archives, documents, notes, downloads, and appointment access.

  1. Open the active file from the relevant strata record to review file number, address, property types, and fiscal-year details.
  2. Use the Active tab to work with current survey questions and answers, and use Archive to review overwritten responses with attribution.
  3. Use the Documents tab to review uploads, approve or deny document versions, and confirm when all required items are approved.
  4. Treat internal notes as staff-only records that are visible to administrators, assistants, and inspectors but not clients. Only administrators can delete notes.
  5. Use the download tools to create a ZIP of uploaded documents or a PDF of survey answers when records need to be exported.
  6. Use Offer Appointment when you are ready for the client to access booking availability for half-day, full-day, or draft-meeting scheduling.

User Management

The Users page manages both internal staff and client accounts, including role setup, strata assignment, property-type permissions, and invitation handling.

  1. Open Users from the left navigation.
  2. Use Create New Users when adding administrators, assistants, inspectors, or client users.
  3. For client users, assign the correct strata and then confirm which property types that user should be allowed to see.
  4. Use role labels such as property manager or council member to identify client contacts clearly.
  5. Edit user details whenever needed, but remember that email addresses cannot be changed because they are tied to login identity.
  6. Use Resend Invite if the confirmation email does not arrive.
  7. Delete users cautiously because related answers and inputs are removed, and staff appointments may need reallocation before deletion.

Document Management

The Documents page is the admin document hub for active files. It supports direct uploads, client-submission review, approval workflows, and cleanup when Dropbox and portal records drift out of sync.

  1. Make sure the strata has an active file, because documents attach to the file number rather than to the strata record alone.
  2. When uploading a document, select the correct strata, file number, document type, and property type.
  3. Remember that only users with permission for a property type can see documents in that section.
  4. Review client uploads through the document-review flow, approving or denying versions and saving progress before finalizing.
  5. Remember that admin-uploaded documents are auto-approved.
  6. Use Sync with Dropbox if folders have been changed outside the portal and the system needs to clear orphaned records.
Storage note: Active files appear in the main list, while completed files move to separate storage after the file lifecycle is closed.

Question Filtering

The Questions page is the global question bank for the portal. Changes here affect every strata and file, so it is used for system-wide configuration rather than file-specific adjustments.

  1. Open Questions from the left navigation.
  2. Use Show Sub-Questions when you want to isolate sub-questions, or leave it off to browse parent questions more broadly.
  3. Remember that parent questions require an answer state, while sub-questions are optional and appear conditionally after the parent.
  4. Use Create New Question to add a global question, setting answer type, optional info text, and allowed response rules such as Not Applicable or Not Available.
  5. Add or link sub-questions as needed. Sub-questions inherit the parent property type and category and can be attached to multiple parents.
  6. Edit existing questions when needed, but keep category changes global and use file-specific configuration inside the active-file workflow instead.
  7. Remember that this module is visible to assistants and administrators, but not to inspectors.

System Settings

The System Settings area covers profile maintenance, company holiday calendars, and inspector availability by location and meeting type.

  1. Open System Settings from the admin navigation.
  2. Use the tabs to switch between Profile, Company Holidays, and Inspector Availability.
  3. Update your own profile details or password as needed.
  4. Maintain company holidays carefully, especially when deciding whether a holiday should recur every year or remain a one-time date.
  5. Remember that only administrators can remove holidays.
  6. Use inspector availability to set open dates, available locations, and custom windows for virtual meetings.
  7. When batch adding dates, review weekend and holiday warnings before confirming availability.

Appointment Management

The Appointments page is used for direct appointment creation and ongoing schedule maintenance. It can bypass normal client-facing availability gates when the admin team already has scheduling confirmation.

  1. Use Add New Appointment when administrators need to enter an appointment manually.
  2. Remember that manual appointment entry bypasses inspector-availability and short-notice restrictions because availability is assumed to have been confirmed already.
  3. Edit existing appointments to adjust timing, cancel them, or mark them as completed after the inspection or draft meeting has happened.
  4. When canceling, record the reason so the client sees accurate context when they log in.
  5. Use Request Rebooking to trigger reminder follow-up for clients who have not booked a replacement date.
  6. Switch between calendar and list views depending on whether you are scheduling or auditing history.
Workflow note: Offer Appointment inside strata details opens availability for client booking, while the main Appointments tab creates confirmed appointments directly.

Timeline Management

The Timelines page tracks non-appointment dates tied to active files, such as AGM dates, fiscal-year milestones, and other reporting deadlines.

  1. Use the timelines module to view date items in either list view or calendar view.
  2. Filter by deadline type or active strata, and switch between future-only and full history views as needed.
  3. Remember that some dates are automatic, including file open date, documents finalized date, and finalized survey date. These cannot be edited manually.
  4. Edit manual dates such as AGM or fiscal-year dates when corrections are needed, and expect those updates to be visible to clients.
  5. Use this module for assistants and administrators. Inspectors do not have access.
  6. Remember that timelines tied to a file disappear when that file is closed and are recreated for new files.

Deleting a File

Deleting a file is an end-of-lifecycle administrative action used after report delivery, when the original request must be closed before a new one is created for the same strata.

  1. Before deleting the file, export survey answers as a PDF if the record needs to be preserved.
  2. Understand that deleting the file removes active and archived answers, notes, and document states such as Not Received or Not Applicable.
  3. Visible documents are moved to completed Dropbox storage for compliance, helping keep old and new file cycles separate.
  4. After deletion, the strata returns to an inactive state with no active file number and disappears from the default list until Show Inactive is enabled.
  5. Treat this as an administrator-only action and use it only when the file is fully complete.
Storage note: Completed document folders are tagged separately so older request records do not mix with future files for the same strata.
