1. planning-with-files

Project planning Persistent context

Pain point: 做项目做到一半忘了全局 — you get halfway through a project and lose the overall picture.

What it is

planning-with-files is a GitHub repo described as a Claude Code plugin that uses persistent markdown files for planning, progress tracking, and knowledge storage. The core idea is to keep project state in files so the agent can repeatedly return to the same roadmap instead of “forgetting” structure between steps.

How to use it
  • Put your goals, milestones, notes, and progress into markdown files.
  • Ask the agent to read those files and rebuild the project roadmap.
  • Use it for task decomposition, tracking what is done, and deciding what comes next.
  • Useful for long-running builds, feature work, research projects, and multi-step debugging.
Good use cases
  • Large coding projects with many unfinished pieces
  • Turning scattered notes into an execution plan
  • Keeping a running project memory in markdown

2. superpowers

Developer workflow Agentic framework

Pain point: 缺少完整的开发工具链 — you do not have a complete development workflow/toolchain.

What it is

superpowers is a GitHub repository that presents itself as an agentic skills framework and software development methodology. It is meant to give coding agents a more complete workflow by combining composable skills with operating instructions.

How to use it
  • Install it when you want an AI coding agent to follow a more structured engineering process.
  • Use it to connect planning, implementation, code review, writing, and other repeated software tasks.
  • Good for developers who want the agent to behave less like a one-off chatbot and more like a workflow-driven assistant.
Good use cases
  • End-to-end feature development
  • Code review and response cycles
  • Standardizing how your agent works across projects
GitHub

obra/superpowers

3. frontend-design

UI design Official Anthropic skill

Pain point: AI做的前端一眼就看出是AI做的 — the front end looks obviously AI-generated.

What it is

frontend-design appears in Anthropic’s public skills repository. Its description says it is for creating distinctive, production-grade frontend interfaces with high design quality and for avoiding generic AI aesthetics.

How to use it
  • Give it HTML/CSS/JS, a component brief, or a screenshot.
  • Ask it to improve spacing, hierarchy, typography, layout, and polish.
  • Use it for landing pages, dashboards, React components, visual redesigns, and beautifying rough UI output.
Good use cases
  • Making AI-generated UIs feel more intentional
  • Improving visual consistency across a web app
  • Polishing a page before demo or deployment

4. notebooklm

Research workflow NotebookLM integration

Pain point: 写东西切窗口查资料 — while writing, you keep switching windows to look up sources.

What it is

Public GitHub repos for notebooklm-skill describe it as a way for an agent to query Google NotebookLM notebooks directly and get source-grounded answers from your uploaded documents. That makes it useful for writing with references close at hand.

How to use it
  • Put your source documents into NotebookLM.
  • Use the skill to query those notebooks from your coding/writing agent.
  • Ask for summaries, evidence extraction, comparison, and citation-backed notes while drafting.
Good use cases
  • Essay or report writing with many source documents
  • Turning large document sets into outlines
  • Research-heavy content where hallucination reduction matters
GitHub

PleasePrompto/notebooklm-skill

5. baoyu-skills

Content pipeline Skill collection

Pain point: 内容从写到发布中间环节太碎 — the path from writing content to publishing it is too fragmented.

What it is

baoyu-skills is a GitHub repository containing multiple skills rather than one single narrow feature. Publicly visible examples include article illustration and WeChat publishing workflows, which suggests it is aimed at content creation, enhancement, and publishing automation.

How to use it
  • Use it when your content workflow has too many separate steps.
  • Draft content, add supporting visuals, transform it into publication-friendly output, and publish through connected workflows.
  • Especially useful for Chinese-language content operations and platform publishing flows.
Good use cases
  • From draft to formatted article
  • Generating illustrations for articles
  • Publishing to channels such as WeChat

6. webapp-testing

Testing Official Anthropic skill

Pain point: 做完网页懒得一页页检查 — after building a site, you do not want to manually check every page one by one.

What it is

webapp-testing appears in Anthropic’s public skills repo and is described as a toolkit for interacting with and testing local web applications using Playwright. It supports checking frontend behavior, browser logs, screenshots, and debugging.

How to use it
  • Run it after building or updating a web app.
  • Use it to test main user flows like login, forms, navigation, and responsive behavior.
  • Capture screenshots and inspect logs when a page does not behave as expected.
  • Use it as a structured QA layer instead of random manual clicking.
Good use cases
  • Regression checks after UI changes
  • Smoke tests for multiple pages
  • Browser-based debugging with Playwright

7. skill-creator

Skill authoring Official Anthropic skill

Pain point: 想创建Skill但不知道从哪开始 — you want to create a skill but do not know where to begin.

What it is

skill-creator appears in Anthropic’s public skills repository and is described as a skill for creating new skills, improving existing ones, running evals, benchmarking performance, and optimizing descriptions so skills trigger more accurately.

How to use it
  • Start from a repeated task you want your agent to handle better.
  • Define the trigger, inputs, expected outputs, and reusable steps.
  • Use it to draft the skill structure and improve it through testing and iteration.
  • Use it again later to refine descriptions, evals, and reuse patterns.
Good use cases
  • Building your first reusable skill
  • Converting ad hoc prompts into repeatable workflows
  • Improving a skill that triggers poorly or gives inconsistent output
